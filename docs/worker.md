# バックグラウンドジョブ仕様

## 概要

`internal/worker` パッケージがバックグラウンドジョブを管理する。メインgoroutineとは別のgoroutineで動作し、`context.Context` によるキャンセルでGraceful Shutdownに対応。

---

## SyncChannel（チャンネル同期）

### 実行間隔

30秒ごと（起動直後にも1回実行）

### 処理フロー

```
1. DBから現在配信中のチャンネル一覧を取得
2. PeerCastデーモンの /admin?cmd=viewxml を叩いてXMLを取得
3. XMLレスポンスの各チャンネルについて：
   a. ジャンル文字列をパース（YPプレフィックスが一致しなければスキップ）
   b. トラッカーホストを決定（Push以外で、Tracker=trueを優先）
   c. DBのチャンネルと照合（CIDベース）
      - 既存 → 更新
      - 新規 → 作成
4. XMLに含まれなくなったチャンネルの is_playing を false に更新
5. アクティブなチャンネル一覧をキャッシュに保存
6. 10分間隔のログ記録タイミングであればチャンネルログを記録
```

### ジャンル文字列の書式

PeerCastのジャンル文字列は以下の書式を持つ：

```
{YPプレフィックス}[{ネームスペース}:][?][@...]{ジャンル文字列}
```

| 要素 | 説明 | 例 |
|---|---|---|
| YPプレフィックス | YP識別子（設定値、デフォルト `ap`） | `ap` |
| ネームスペース | 省略可。`:`区切り | `ns:` |
| `?` | リスナー数非表示フラグ | `?` |
| `@` | 帯域制限（`@`=FW、`@@`=BW、`@@@`=HBW） | `@@` |
| ジャンル文字列 | 任意のジャンルテキスト | `ゲーム` |

例: `apns:?@ゲーム` → プレフィックス=ap, NS=ns, 非表示=true, 制限=FW, ジャンル=ゲーム

### ログ記録の条件

- 現在時刻を10分単位に切り捨て（`time.Now().Truncate(10 * time.Minute)`）
- 前回記録時刻と異なる場合のみ記録（メモリで前回時刻を保持）

## DailySummary（日次集計）

### 実行タイミング

毎日 0:03

### 処理フロー

```
1. 前日（24時間前）の日付を算出
2. その日の全チャンネルログをDBから取得（名前・時刻順）
3. チャンネル名ごとに集計：
   - ログ数（記録回数）
   - 最大リスナー数
   - 平均リスナー数
4. 集計結果を channel_daily_summaries テーブルに保存
```

---

## キャッシュ戦略

| キー | 内容 | 有効期限 | 更新タイミング |
|---|---|---|---|
| `channels` | 配信中チャンネル一覧 | 5分（デフォルト） | SyncChannel 実行ごと |
| `information` | お知らせ情報一覧 | 5分（デフォルト） | index.txt リクエスト時にキャッシュミスで取得 |

キャッシュミス時はDBにフォールバックする。
